# 腾讯、石墨、金山在线文档简单对比 - 知乎

所做工作的产品，除了图纸表格，就是成册的文档。而一般少则三四个人，多则十几号人参与一本文档的编写。目前公司工作流程为，每个人用 Word 或 WPS 写完各自负责的部分，然后通过 QQ、微信或邮件发给项目经理汇总成册。汇总阶段对于项目经理就是重体力活，要把 N 个 docx 文件合并成一个，还要统一排版调格式。排版是最头疼且消耗耐心的，因为：
・即使提前做了统一的格式规定，还是有人会按照自己的写作习惯。这是人的不确定性；
・从一个 docx 文件拷贝到另一个 docx 文件中的内容，即使选择保留原格式粘贴，有时候格式还会神奇改变，尤其上下角标和列表。这是工具的不确定性；
・MS Word 与 WPS 之间的格式不兼容（目前两者之间已经差异甚小）。
于是就想能不能用在线文档工具实现协同写作，十几号人用同样的工具编辑同一个文件，格式更容易统一，也不存在合并文件这一步骤。
再进一步考虑本公司情况，如果跨部门的十几号人协同在线写作，然后完成一本文档的初稿、校审、终稿以及打印，那么这个在线文档工具至少需要同时满足以下条件：
1.能导入导出 docx 格式文件；
2.对于 docx 文件的排版格式要与电脑上的 MS Word 或 WPS 的显示相一致（总有人不会或不愿使用在线工具，尤其老员工或领导以及甲方）；
3.编写文档报告所必要的文字编辑功能，比如标题等级、字体大小、段落设置、插入图片表格等；
4.具备协同功能，能处理十几号人编辑一个文件所产生的内容冲突；
5.在线工具必须加载快且响应迅速；
6.能导出 PDF（以防万一导出的 docx 文件排版垮了又着急发出）
