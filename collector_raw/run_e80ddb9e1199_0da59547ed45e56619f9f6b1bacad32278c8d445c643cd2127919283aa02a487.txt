1楼 [2021年11月16日](https://chaoli.club/index.php/conversation/post/66015 "2021-11-16 20:18:49")   本人在办公室从事文字处理工作2年，高强度使用WPS，也带着用office，我简单梳理一下我这两年遇到的兼容性问题。

超理吧有很多人是有数学公式输入求的，这个我不深入研究了，我主要还是探讨一下正常的文档办公场景。

1.有人吐槽WPS写文章，插入的图片用Word打开，排版会发生变化，所以尽量用Word写。

我讲一下我的经验，一般来说，插入的图片用默认的格式，不去改版式，两边正常兼容，不兼容的情形主要是采用了浮动版式，这样一来，图片的“相对位置”就没有办法固定了。有什么绝佳的兼容方法呢？插入一个表格，然后图片放在表格里 ，这样图片的相对位置就固定了。

2.Word中的邮件合并功能可以直接读Excel的xls文档作为数据库，但WPS得用WPS的格式et作为数据库，两者不兼容。

3.在Excel中，如果使用超规格的字号（72号为界限），超过72号的字体继续放大，在WPS中是无法识别的，仅仅能识别72号。反之亦然。 在Excel中用这么大的字有什么用呢？打姓名牌用的。这个比Word好用一些。这个超规格内容在Word中也有体现，我设置页边距为0.5，然后让图片尽量在Word中展开，这样的“极限”类型的文档两边显示不一致。

4.在Excel中的缩放打印，和WPS也是不兼容的。

5.超10万行的表格不兼容。这个10万行是我的经验结论，不是绝对分界线。正常我处理的是6万行数据，两边正常兼容。但是后来处理了30万行数据时，差异就来了。一个csv格式的表格，Excel可以慢慢读出来，但是WPS直接卡壳，实在是都不出来。但是，Excel读出来后，另存为一份xlsx文件后，WPS就可以读了。但反转又来了，如果WPS此时编辑过，Excel也读不出来，变成WPS独享的表格了。

总的来说，文档处理中，不兼容的场景主要是一些很极限的场景，比如100m大小的xlsx文档，满是图片但又自由移动的图片docx文档。

最后再讲一下我使用哪些功能。

Excel表格方面，函数主要是正常的加减乘除，vlookup，mid，mod，if，iferror，countifs，sum，avg，concentrate，&，text，datedif，len，left，right，主要是一些逻辑、文本、日期时间、查找引用类函数。

Word中主要是用字体、段落、编号、页边距、分页、页眉页脚、修订、比较、文章结构图等功能。文字编辑上，WPS比office要更适合中国人办公习惯，office的编号让我非常痛苦，写论文的那种1.\1.1\1.1.1，还有缩进问题。用一、（一）、1. （1）这种在Word上实在是麻烦，改设置教程也不清晰，我摸索了很久才弄明白。

最后来一组自问自答

问：为什么用WPS？

答：因为office的自动保存功能需要联网，断网下自动保存一塌糊涂。WPS可以在断网下进行自动保存。

问：你遇到过WPS闪退问题吗？

答：除非是打开超大的xlsx表格，其他情况并没有明显遇到过。我自认为可能跟内存大小有关系，小内存电脑卡的时候会清退软件。

问：你怎么看WPS有广告，office没有广告？

答：如果都不花钱，没有广告的office除了学生版本，那自然是破解版本，同理，你也可以装破解WPS。如果是正版，那WPS花钱开会员也没有广告。当然，office有合购方法，折合价格比WPS便宜。

几点感想：

    1.   MS Word 前几年不时遇到几个很头疼的问题：模板坏掉，整个文档列表项缩进很多地方变黑块。WPS 文字没遇到过类似的问题。（也许就是这几年都用得少了）。
    2.   1202 了，Excel 能打开同名工作簿了吗（
    3.   XLSX 之类的互操作问题可以算 ISO/IEC 29500 的符合性问题，不只是具体实现之间不兼容。
    4.   至于 CSV 辣么简单的格式都能卡就是实现叒鸡。
    5.   WPS 原生文档基于微软过气格式 Composite Document File V2 ，倒是 MS Office 现在都不认了……
    6.   应用闪退那就是 bug 。真要挂了，也得通知用户在界面上好歹留个尸体。要是操作系统闷声不响杀进程，那是系统 bug ，至少有用户界面的前台应用就不应该突然挂掉，干掉也同样应该要告诉用户挂了。

对 Windows 来说，整个系统不存在内存不足闪退问题，因为内存真不足整个 Win32 界面首先会出毛病，GDI 抽风原生控件没字，注销都救不了的那种。

Linux 内核 OOM 倒是可能随机乱杀进程，但那时候基本 DE/WM 也挂得差不多了。

搞笑的是 MS Office 的几个应用（另外还有 Visual Studio ）我都遇过“内存不足”的报错弹框（对话框里面就一个“内存不足。”，然后一个确认键，什么上下文信息都没有），然后可能还会有可视的界面异常（某些地方缺字）以及无法撤销编辑无法保存等，但实际上系统内存多得很（也不是进程地址空间的问题）。如果是内存碎片太多导致分配失败那是实现辣鸡，而且也不应该报这样的错；这种乱报错也明显是 bug 。估计不是对象池实现辣鸡就是八成是哪的 COM 调用抽风了。
