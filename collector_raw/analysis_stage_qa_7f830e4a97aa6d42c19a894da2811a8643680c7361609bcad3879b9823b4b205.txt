# 对话钉钉 CTO:从低代码到酷应用,To B、To C 技术大融合的必然趋势 - 知乎

欧亚板块和印度洋板块的剧烈碰撞，形成了世界上最大的弧形山脉喜马拉雅山脉及巍峨的珠穆朗玛峰；当数字化走进深水区，组织治理及业务运营理念持续升级背景下，To C 与 To B 技术的碰撞与融合，也同样有机会成就出全新一代的企业级应用。

“在技术人群的印象里，To C 的 高并发、高可用及高性能，代表了一代技术潮流；传统 To B 软件体系，则更侧重做系统化顶层设计，强调高度灵活的可配置性，解决的是组织和业务如何随需应变。最近一年多，钉钉从组织数字化走向业务数字化，强调 PaaS First、Partner First。在这样的战略驱动下，我们越来越感受到，To B 与 To C 技术在加速碰撞和融合，这种冲撞也在塑造中国软件业的新物种。”钉钉 CTO 程操红认为。

比如酷应用。今年 3 月底，钉钉推出了一款全新应用模式。它将普通应用的多种功能拆解后融入在不同的场景中，成为一种全新的应用形态。钉钉之前提出的“ 低代码革命”是对 To B 应用开发方式的一次革新，而“酷应用”则是对 To B 应用使用方式的一次革新。

据程操红介绍，从技术竞争力看，钉钉首先有强大的企业级 IM 平台，不管是从规模、成本、组织沟通特点、还是开放性，满足在线高效协同沟通各方面的需求；

除了 IM 平台，钉钉还提供丰富的协作基础产品和能力，包括文档、钉闪会、音视频和 Teambition 等等；

此外，钉钉去年初大力推动低代码，推出钉钉搭、成立低代码联盟，也让钉钉成为应用开发平台。在疫情带动数字化需求爆发后，将数字化能力普惠给更多业务人员，有效解决组织内外部广泛的数字化需求，让更多内部业务在线协同起来、数据通起来；

最后是酷应用，如果说低代码解决的是应用的快速搭建问题，那么酷应用解决的就是应用的活跃问题，让应用真正在组织内和组织间能用起来。
To C、To B 技术的大碰撞与大融合
技术的碰撞与融合，作用到用户中，势必会打破原有的 To B 软件形态，形成新的产品与技术形态，酷应用便是其中的产物。

传统面向 To B 的 ERP、EHR、CRM、MES 等软件，其本身都是基于企业管理理念来进行构建，采用的是一种自上而下的视角。拿 ERP 软件举例，通常员工进入软件后会看到一个整体的功能菜单，包含从管理员侧的基础数据设置、权限角色设置、流程及规则设置到诸如财务管理、库存管理、销售管理等业务域各类复杂功能。
