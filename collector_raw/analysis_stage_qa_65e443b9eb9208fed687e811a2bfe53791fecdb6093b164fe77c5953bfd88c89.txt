# 一、国家积极推进制造业高质量发展，数智创新推动制造业数字化转型

# 二、制造业数字化转型趋势分析 

消费互联网与产业互联网的双轮驱动 

如何看待数字化转型的成本与收益，将决定制造企业推进数字化转型的力度、路径和方式。过去， 衡量数字化转型的直接产出是困难的。制造企业的信息化投入很难直接反映为生产率的提升，但又很难 忽略数字化转型对企业创新和行业发展的外溢效应。而消费互联网与产业互联网的结合，为制造业数字 化转型提供了一个更为清晰的发展路径。因为推进数字化的本质目的，是通过海量数据的沉淀和建模， 更为精准地理解事物运转过程和深层机理，降低经验判断的误差和不确定性，从而变革企业组织和生产 流程。消费互联网与产业互联网的双轮驱动，让制造企业更加明晰数据要素的生产价值，令企业生产模 式从面向库存生产往面向订单生产迁移，同时伴随有 C2M 定制需求，加速业务的持续创新。此时，如何 让业务团队具备基础开发能力至关重要，企业数字化需求也会迭代加速，等不及标准软件迭代，利用低 代码甚至无代码构建系统的能力成为刚需。 

流程领域有望率先实现智能化 

工作中每件事都可以被数字化、智能化，但需要找准切入口。流程领域的生产本质上是连续的，生 产工艺相对简单，生产流程清晰连贯，因此生产全过程数字化难度相对较低。这是数字化的加速引擎。 生产流程数字化，加大工作的透明化程度，透明化的工作提供了流程再造的机会，提升业务效率；效率 的提升， 沉淀了更多的数据， 带来了数据辅助业务智能决策的机会。 在全面贯通整合各阶段数据的基础上， 流程领域企业接下来应多加运用机器学习、自然语言处理及计算机视觉等人工智能技术进行实时数据分 析和实时决策，并进一步将智能系统延伸至供应链、生产后服务等各个环节，最终实现全面智能化。 

着力建设智能车间，强化集成技术应用 

智能制造最大的特征是对外部环境的变化做出实时的响应， 智能车间是落实智能化改造的重要载体， 是制造业实现智能化转型升级的基础。制造业企业正逐步对车间生产信息进行可视化与数据统计，对业 务计划和任务进行自动排程，完成了业务流程的人、机、料等数据的收集，因此智能车间运作将产生大 量数据。因此，智能车间建设将从单机智能走向系统智能。从技术层面看，这预示着 OT 底层系统的解构， 也将实现 OT 系统与 IT 系统的原生融合。其核心是让变化快的软件摆脱束缚，提供更多个性化的服务； 让利用率高的硬件趋于标准，实现规模经济效益。实现软硬件耦合的关键，在于改变封闭、刚性的技术 组织架构，将之迁移到一个开放的、去中心化的架构体系中，利用弹性的集成技术是实现企业内业务、 数据互联的核心技术，可以帮助企业达到信息和数据的实时交互。 

设备数字化和网络化能力提升，支撑在线质量管控 

智能制造的基础是设备数字化和网络化，核心是系统智能。目前，制造业企业设备数字化率、设备 联网率等稳步提升，设备数字化水平进步显著，数字化系统愈发区域复杂。此时，对制造企业数字化的 考验是，如何解决复杂系统的不确定。而在线质量管控是评价一个企业智能制造水平的关键指标，它是 生产经验沉淀、复用、重构的紧密结合体。利用数理统计方法的过程控制系统，实现对制造业工序的实 钉钉未来智造白皮书 ·04·

# 三、钉钉介绍 

## 3.1 阿里巴巴云钉一体战略简述 

2020 年年底，阿里巴巴提出“云钉一体”战略。在 “云钉一体”战略中，“云”毫无疑问是指阿里 云提供的云基础设施，而“钉”对应的就是钉钉，立足 5 亿用户和 1900 万组织的巨大端口流量，为企业 提供服务。“云钉一体”的本质，就是面向企业、机构甚至个人，提供一整套数字化的“基础平台 + 应 用程序开发体系”，用户可以按照自己的需求，定制云计算服务。 

> 图 云钉一体

为何需要“云钉一体”？ 

2020 年的疫情加快了全社会数字化转型的速度，原本需要三五年的发展过程，在一年之内就完成了 转换。 过去的 10 年，是数字化的 1.0 时代，企业数字化转型第一个阶段是“全面上云”。典型特征是企业 IT 基础设施上云，以互联网用户为主体的领先企业，开始降低设备采购投资，改为把系统构建在云上。 2020 年是应用上云和传统企业上云的 2.0 时代。 在这个时代， 云早已成为数字化时代的关键基础设施， 取代传统的计算和存储方式。但为企业提供服务的应用不够繁荣，企业需要大量低门槛个性化的云原生 的应用，让没有技术的传统行业也能全面上云。这时企业需要一个内部的软件系统平台，集成这些应用， 这个平台需要能解决公司的业务流、信息流、数据流，还要移动化。“云钉一体”在 2.0 时代应运而生。 

如何实现“云钉一体”？ 

阿里云的“云钉一体”战略，将云基础设施与操作系统紧密结合，将成为云计算领域跨时代的里程碑， 钉钉未来智造白皮书 ·06· 

引领数字化转型浪潮的创新步伐。总结起来是三大方向： “做深基础”，就是从飞天云操作系统向下延伸定义硬件，在芯片、数据库、路由器、交换机等领 域加大投入力度，构建整套云基础体系。 “做厚中台”，就是将钉钉这样的新型操作系统与阿里云进行深度融合，实现“云钉一体”。把操 作系统的功能做得更强大，包括 AI 能力、大数据处理能力、IoT 能力、移动能力等。 “做强生态”，就是基于云和新型操作系统，构建一个繁荣的应用服务生态。阿里云加大与行业 ISV 的合作力度，支持行业应用开发，共同打造联合解决方案。 许多企业都已将“云钉一体”视为基础设施，包括立白、东方希望、太平洋保险、广东农信等企业， 已开发数十万种企业应用。企业有跨层级、跨地域、跨系统、跨部门、跨业务的协同管理需求，这些都 以钉钉为入口，并架设在云上，钉钉提供 “基础设施”能力。打个比方就像一条高速公路，云是支撑道 路的路基，路上可以跑各种汽车，也就是说企业组织可以自行搭建或购买应用，不同路段之间的接口也 是统一的，这样才能跑得更快更顺畅。

全球知名咨询公司 Gartner 的数据显示，2021 年对于应用开发的需求将达到所有 IT 公司开发能力的 5 倍。这是企业数字化程度不断提高之后，共同面对的一个尴尬局面，越来越多的业务需求远远超过 IT 开发能力，他们需要完成的项目似乎永无止境。面对这一巨大的供需缺口，低代码、无代码开发几乎是 唯一解决方案。 什么是低代码？低代码开发为开发者提供了一个创建应用软件的开发环境，企业的应用开发可以通 过“拖拉拽”的方式即可完成。它通过高度抽象的表单引擎、视图引擎、流程引擎、规则引擎、报表引 擎及高级宏语言等，用可视化的形式进行设置组合，实现对各种复杂业务系统的快速开发。区别于传统 的软件开发模式，其最大特点是不需要手动编写大量的源代码。 低代码开发平台能够实现业务应用的快速交付，早在 2016 年，Forrester 调研结果显示，大部分公 司反馈低代码平台帮助他们把开发效率提升了 5-10 倍。低代码也能够降低业务应用的开发成本，低代码 开发还显著降低了开发人员的使用门槛，非专业开发者经过简单的 IT 基础培训就能快速上岗。低代码可 以降低技术门槛，让不懂技术的业务实操人可以自己搭建应用模型，也让企业自己掌握更多数字化转型 的主动权。 基于云钉一体的“低代码开发”，将成为新一代的软件开发方式。钉钉的低代码开发平台都基于云 服务提供，开发者不要直接在云上开发，而是通过钉钉这样的应用平台来开发，开发的方式就是低代码 和无代码，企业应用天然具备云化、移动化、数据互通性。目前在钉钉上提供低代码的服务不局限于宜 搭氚云等低代码开发工具，很多专业场景的 SaaS 应用，都包含了方案定制化和个性化的平台能力，支持 业务自定义和配置实现。 作为企业应用开发平台，钉钉全面开放，截止至 2021 年 9 月份，钉钉基于底层能力已经开放 2000 多个 API 接口，方便合作伙伴与客户将自身业务应用基于钉钉 API 接口在钉钉上进行构建，常用的开放 能力包括钉钉通讯录、场景群、群机器人、工作台等。

# 四、钉钉制造行业解决方案 

## 4.1 制造业企业面临的挑战 

受到国际局势及疫情影响，制造业企业竞争环境发生急剧转变，让企业面临内外双重挑战，从企业 外部来看，个性化、定制化等需求带来了对制造的变化，多品种多批量、高质量低成本、柔性生产等， 同时产品同质化带来了行业内激烈的竞争，供应商交期管理与质量管控问题日益突出；从企业内部看， 制造型企业面临招工难、留人难，劳动力、材料、库存成本不断攀升，车间管理、质量稳定性差，核心 技术弱创新能力不足等诸多问题。制造企业纷纷寻找良方，智能制造、企业数字化、产业链数字化等成 为热词。企业在转型的实践过程中也留下了众多问题，其典型问题如下： 1. 业务快速发展，数字化系统众多，每上一个系统就需要安装一个新的 APP，一套独立的账密系统， 员工操作麻烦复杂，企业缺少一个能够面向所有员工的统一窗口； 2. 设备不在线，设备异常状况获取被动，问题处理协同效率低下，影响维修进度进而影响产能； 3. 业务的快速发展要求数字化系统具有较快响应能力，传统标准数字化产品迭代难以快速响应企业 实际需求问题； 4. 盲目上系统带来的平台割裂、数据孤岛问题，让业务数据难以沉淀分析； 5. 企业数字化快速发展过程中，企业信息安全往往容易忽视，过程缺少规划，后果往往致命； 6. 产业链协同依旧业务与沟通分开，效率低、人员身份难以保障等问题。 虽然企业对数字化是拥抱的，但仍然普遍存在想用不敢用、不会用、用不起的问题。如何更好的应 对这些挑战成为摆在制造企业面前的最为现实的问题。

四川德恩精工科技股份有限公司是创业板上市公司， 国家 “专精特新” 小巨人企业、 国家高新技术企业。 公司成立于 2003 年，经过十多年的创新发展，拥有员工 1500 余人，其中研发人员 160 余人，公司已成 为国内知名的机械传动零部件制造商，中国带传动细分行业的龙头企业。2018 年起，德恩精工开始智能 制造转型升级，联合阿里云积极探索“互联网 + 先进制造业”工业互联网应用新模式，推动传统制造业 向现代服务业深度融合。 

多层级的组织协同平台 

各分子公司及工厂之间的办公协同以前都比较分散，通过各种社交工具进行沟通，内部工作信息分 散无法保障安全性。 钉钉未来智造白皮书 ·49· 

2018 年开始通过使用钉钉的审批流，现在更深入地使用专业版高级审批，并且全面统一集团内工作 的沟通工具，确保内部业务沟通的统一和安全。同时采用钉钉作为统一门户和流程中心，全面承接集团 内部各项行政、人事、财务等审批，高效解决跨组织、跨地域的业务协同，大大提升工作和沟通效率及 安全性。 

高效实现精准人效管理 

集团人员岗位复杂，销售、工人、职能岗位的员工都有不同的考勤要求，高效准确地掌握全集团人 员的出勤管理，曾经更多地是采用线下统计和层层上报，统计过程复杂，真实性和准确性较差。 通过使用钉钉的组织、 人员、 考勤管理等功能， 通过钉钉移动打卡、 B1 人脸识别等软硬一体解决方案， 实现集团内各岗位的复杂考勤管理。同时将考勤数据与企业 ERP 的人力管理数据结合，实现自动计算全 员各岗位的人员薪酬，从而高效、准确的实现人工成本及人效的分析数据，支持生产经营决策。 

数字化项目管理平台，支撑精细化项目交付 

公司产品工艺流程比较复杂，产品规格型号和客户数量众多，订单呈现多品种、定制化的特点，定 制化项目交付过程中容易出现项目信息同步慢、关键节点管不到等问题，导致订单逾期时有发生。此外， 机加工行业的经验传承主要依赖老师傅传帮带，项目图纸、工艺文档等公司资产难以统一归档和沉淀。 德恩精工通过钉钉自有产品 Teambition 将定制化交付项目流程梳理并实现线上化，每个阶段拆解出 规定动作分配到人和时间，实现项目交付过程的标准化、精细化和过程可视化管理，从而减少逾期率。 每个项目自带项目级知识库，将每个项目的过程信息、执行记录、文件交付物统一归档和管理，实现经 验“数字化”传承。 钉钉未来智造白皮书 ·50· 

质量流程可视化，提高问题处理效率 

客户反馈的质量问题通过邮件传递，信息反复确认、问题处理状态不清楚，效率较低，且难以对质 量问题进行数据化统计和度量。 通过 Teambition 搭建问题处理流程看板，将处理过程可视化，每个问题（卡片）的处理节点、责任 人、处理时间、处理进度、逾期情况非常清楚，标准的处理流程确保了流程运作的有效性和沟通的效率， 实现质量问题「反馈 - 分析 - 跟进 - 关闭」的闭环管理和质量统计分析。

内部管理数字化，让管理效率不断提升 

以前，公司内信息分散，流程和沟通混杂。工作任务下达、审批或报批都是通过书面签字方式进行， 每次纸质审批都要通过几个部门才能完成，如果领导不在，可能还要等上几天。这样不透明方式带来的 钉钉未来智造白皮书 ·51· 

就是流程不流畅，职责不清晰，任务也没法精准传达，效率非常低。 现在，通过钉钉实现业务流程线上化后，不同的业务流程设计相应的线上流程，节点明确，审批效 率大大提高，以前要几天的审批，现在几分钟就可以完成。再不用每个部门沟通协调，真正实现了内部 管理的数字化。 

销售、生产、财务等跨部门高效协同 

以前， 公司生产环节有 MES、 ERP 系统， 内部管理有 OA 系统， 每个系统并不互通， 各自有不同的解决方案， 运算出的大量数据需要员工手动录入，十分繁琐。没有统一的平台对接入口，切换平台需要单独登录， 也没有通用接口，专门打包 API 和 SDK 对接，多个异构系统对接后，出现问题也难以跟进，找到问题的 关键。 现在，组织在线、人事管理、员工内训、生产业务等系统集成钉钉后，切换方便，系统数据完成对接， 有效改变过去各业务部门分散业务系统带来的业务孤岛、数据孤岛的问题。在钉钉输入数据后自动同步 到各业务系统，不用各个系统一一维护，减少数据错误率，也方便排查系统运行过程中出现的错误。 对系统运行过程中产生的数据进行分析，结合算法模型，可以清晰对比各项数值，驱动产品迭代升 级优化，加强公司竞争力，也为公司未来的发展提供了依据。 钉钉未来智造白皮书 ·52·
