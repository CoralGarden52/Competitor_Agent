### What's new in Lark CLI this week (May 18–22)

> Lark CLI agents picked up a new batch of capabilities this week. Update to the latest version to try them out.

Get started with one command: `lark-cli update`

* * *

**Markdown files: in-place edits, version diffs, direct-to-Wiki creation**

*   **Patch a section in place** (`markdown +patch`): Update one section without overwriting the rest of the file.
*   **Diff two versions** (`markdown +diff`): Compare two cloud versions, or compare a cloud version against a local `.md` file.
*   **Create directly under a Wiki node** (`markdown +create --wiki-token`): Skip the old "create in Drive, then move" two-step.

**Drive: manage files like Git**

*   **Two-way sync** (`drive +sync`): Local edits upload to Drive automatically; remote edits download back to local. No more manual upload and download.
*   **Full version history** (`drive +version-*`): List, download, restore, and delete versions — four actions in one place.
*   **Inspect a document link** (`drive +inspect`): Pass any Lark Doc or Wiki link; the agent returns the file type, title, and canonical address. Wiki links auto-resolve to the underlying document.

**Base: batch attachments and form auto-fill**

*   **Batch upload, download, and delete attachments** (`base +record-*-attachment`): Up to 50 files per call, 2 GB each. Files larger than 20 MB chunk automatically.
*   **Fetch form fields and submit in one step** (`base +form-*`): The agent reads form fields, fills them in, and submits. Attachments upload automatically.

**Wiki: node lifecycle plus new spaces**

*   **Create, list, and delete nodes** (`wiki nodes create/list` + `wiki +node-delete`): Reorganize a Wiki at scale — group topics, restructure the node tree, and clean up obsolete pages.
*   **Create a new Wiki space** (`wiki spaces create`): Spin up a fresh workspace for a new project in one command.

**Mail drafts: priority visible at a glance**

*   **See draft priority on read**: When the agent reads a draft, it sees the priority flag — no more accidentally sending an "Urgent" draft as a normal email, or vice versa.

**Want to try it?** Start with one command: `npx @larksuite/cli@latest install`

### What's new in Lark CLI this week (May 9–15)?

We've shipped 6 more versions (v1.0.27 → v1.0.32) in just one week. Here's what your agents can now do — update to the latest version to try them.

Get started with a single command: `lark-cli update`

**I. Easier Wiki management for agents**

v1.0.32 introduces three node-management actions in Lark Wiki:

1.   Agents can **list all Wiki spaces accessible to the current account** — useful for scoping cross-space operations.
2.   Agents can **list child nodes under a specified space or node** — pair with search to locate target nodes.
3.   Agents can **copy a single Wiki node to a target location** — supports specifying a new title in the request.

Agents now have the core actions they need inside Wiki.

**II. IM upgrades — chat structure, bookmarks, and finding groups**

Three improvements that bring agents closer to behaving like real teammates in chats.

1.   **Agents can create topic chats directly** — previously agents could only create normal group chats; now topic chats are a single command away.
2.   **Agents can bookmark messages or threads** — view bookmarked items and remove bookmarks — useful for saving important messages for later review.
3.   **Agents can filter out muted chats** when searching or listing groups — find what you need without scrolling past hundreds of muted ones.

**III. Drive sync gets a smart mode**

For scheduled large-directory syncs and archiving CI artifacts to Drive, two improvements landed this week:

1.   **Drive sync now supports a smart mode** that uses modification time to automatically skip unchanged files — large batch syncs of thousands of files run significantly faster.
2.   **Agents can choose how to handle name conflicts when syncing Drive**: fail with a notice, keep the newest, or keep the oldest — conflicts no longer block sync.

**IV. Approvals: add approver and send back for revision**

Agents used to be limited to approve / reject / transfer. A commonly requested actions is now available:

1.   **Agents can add approvers and send approval tasks back for revision** — multi-step enterprise approval flows now run end-to-end through agents.

Use cases: mid-flow correction of multi-approver chains, send-back-for-revision scenarios.

### What new things can your agent do with Lark CLI this past month?

One month after Lark CLI went open source, more than 100 new capabilities are now available — all live. To try them, update your Lark CLI to the latest version. [Get started](https://open.larkoffice.com/document/mcp_open_tools/feishu-cli-let-ai-actually-do-your-work-in-feishu)

### I. Agents collaborating in Lark

One agent doesn't have to do everything. Specialized agents can hand off work to each other in a group chat — and you can watch, step in, interrupt, or course-correct anytime.

1.   **Agents can @ other agents and invoke them directly — the basic channel for multi-agent collaboration is now in place.**
2.   Agents can recognize other bots' names and identities in a group — groundwork for multi-agent coordination.

See the [Lark Agent @ Agent setup guide](https://open.larkoffice.com/document/mcp_open_tools/integrating-agents-with-feishu/agent-at-agent-for-complex-tasks) for details.

### II. Agents send and receive messages like a person

Ask your agent to send a message, and it acts on your behalf — the sender shows your name, the message belongs to you, and permissions follow you. Add or remove group members without switching back and forth between Lark and the agent.

1.   Agents can send messages **as you** — coworkers see your name, not a bot avatar.
2.   Agents can send images, files, audio, and video as you — not just text.
3.   Agents can create groups as you — you become the owner directly, no manual transfer needed.
4.   Agents can add or remove group members in one step — no digging through group settings.
5.   Agents can react with emoji — quick acknowledgments, thanks, or agreement in one tap.
6.   Agents can update group names and descriptions — handy when a project gets renamed.
7.   Agents can fetch message details in bulk (including sender and thread info) — pull dozens at once.
8.   Agents can listen for new messages in real time — respond the moment a message arrives, no polling.
9.   Agents can generate message links — jump straight to the original message.

### III. Agents produce real work — docs, slides, sheets, whiteboards

Ask your agent to "draft a project plan" and you'll find a Lark Cloud Doc already created, formatted, and permissioned — ready to open and edit. The same goes for slides, sheets, and whiteboards.

**Cloud Docs**

1.   Agents can draft and edit Lark Cloud Docs — write in Markdown, and it converts to Lark format automatically.
2.   Agents can replace by section, search by keyword, or insert by paragraph — precise edits without rewriting the whole doc.
3.   Agents can paste screenshots from the clipboard directly into a doc — no need to save locally first.
4.   Agents can insert images and files **anywhere in a doc** — not just at the end.
5.   Agents can upload media files of any size — large videos work too, with automatic chunked upload.
6.   Docs created by your agent open right away — permissions are granted automatically.
7.   Agents can preview images and media inside a doc — take a look before deciding what to change.
8.   When inserting files, agents support three display styles — card, preview, or inline.
9.   **Agents × document comments** — agents can comment on docs, sheets, slides, and Base tables, and respond when you @-mention them in a comment. Your agent can review, suggest, and respond like a teammate.

**Whiteboard**

1.   Agents can generate editable architecture and flow diagrams right inside a doc — live diagrams, not screenshots, so you can drag and edit.
2.   Agents can export whiteboards as images or Mermaid/PlantUML code — embed them wherever you like.
3.   Agents can drop images onto a whiteboard — screenshots and photos go straight in.

**Slides**

1.   **Agents have 42 official slide templates covering 8 scenarios** — no more plain black-on-white decks.
2.   Agents can edit a single element in a slide — change a title without redoing the whole page.
3.   Agents can drop local images straight into a slide — no need to upload to Drive first.

**Base**

1.   Agents can write or update records in bulk — load hundreds of rows at once.
2.   Agents can return only the fields you need — saves tokens and keeps results readable.
3.   Agents can lay out dashboards and add Markdown notes — team-ready reports come together in one step.
4.   Agents can download attachments in bulk — no clicking through them one by one.
5.   Agents can export a Base to a `.base` file for backup, and import it back — **move data without losing anything.**
6.   Agents can generate share links for records, one at a time or in bulk.
7.   Bases created or copied by your agent are ready to use right away — permissions are set automatically.
8.   Agents can build a form with questions in one step — event signups, user feedback, satisfaction surveys, ready as soon as you ask.
9.   Agents can return Base data in Markdown — easier for LLMs to read.
10.   Agents can manage Base workflows — notify and update automatically when records change.

**Sheets**

1.   Agents can read, write, and append cells — Excel-style operations on Lark Sheets in one step.
2.   Agents can insert images into cells — quick visualization without extra steps.
3.   Agents can place floating images anywhere in a sheet — not tied to a single cell.
4.   Agents can add or delete rows and columns, merge or split cells, find and replace, and apply styles — everyday sheet operations covered.
5.   Agents can manage dropdown options in bulk — no adding dozens one at a time.
6.   Agents can manage worksheets — create, copy, delete, and rename in one command.

**Drive**

1.   Agents can upload and download files, create folders, and set permissions — Drive management in one command.
2.   Agents can upload files of **any size** — files over 20MB use chunked upload automatically.
3.   Agents detect file extensions when downloading — files open directly, no manual renaming.
4.   Agents can import, export, convert, and move files in one step — no chaining three separate operations.
5.   Agents can rename files in bulk and create shortcuts — good news for anyone who likes a tidy Drive.
6.   Agents can clean up unused files in one go — no manual review.
7.   **Agents can sync between local folders and Drive in both directions** (push / pull / status) — manage docs the same way Git manages code.
8.   Agents can create, download, and edit Markdown files in Drive — plain-text workflows fit right in.
9.   Agents can upload directly to a Wiki node — files are filed in place, no manual move.
10.   Agents can request access from a doc's owner — let the agent handle permissions for you.

**Wiki**

1.   Agents can read Wikis and create or reorganize nodes — Wiki maintenance runs on its own.
2.   Agents can create and delete Wiki spaces — new team? Set up a Wiki in one sentence.
3.   Agents can add or remove Wiki members in bulk — no click-by-click onboarding.
4.   Agents automatically have permission to Wiki nodes they create — you won't end up locked out of what you just made.

### IV. Agents help you run meetings — from attendance to notes to follow-ups

After a one-hour meeting, open Lark and the notes are written, action items are created, and next week's follow-up is already on the calendar.

**Minutes / Video Conferencing**

1.   Agents can read Minutes transcripts and pull out decisions and action items — no more manual writeup.
2.   Agents can summarize all your meeting notes into a structured weekly report in one command — ten meetings a week, no problem.
3.   **Agents can turn any audio or video file into a Minutes record** — voice recorders, Zoom, phone recordings, not just Lark meetings.
4.   Agents can pull AI-generated artifacts from Minutes — summaries, action items, and chapters, ready to use.
5.   Agents can download Minutes audio and video in one step.
6.   Agents can find the Minutes record from a calendar event or meeting ID — no scrolling through the Minutes list.

**Calendar**

1.   **Agents can check everyone's calendar, factor in time zones, recommend time slots, and schedule the meeting in one step** — when no one's free, they suggest the slot most likely to work and explain why.
2.   Agents can update events — title, time, description, attendees, and meeting rooms — without recreating them.
3.   Agents can search events and check multiple meeting rooms at once — find an open room without clicking through them one by one.
4.   Agents can get event share links — external attendees can join with one click.
5.   Agents can generate a daily summary of your schedule and open tasks — open it in the morning and know what's on for today.

### V. Agents handle your day-to-day — mail, approvals, tasks, OKRs, attendance — no app needed

Your mail, approvals, tasks, OKRs, and attendance live in different places in Lark. Ask your agent in one sentence — it sends mail, approves requests, creates tasks, and checks attendance, without you opening any app.

**Mail**

1.   Agents can scan unread mail, group and summarize it, and draft replies — start your day without going email by email.
2.   When agents reference a local image in a draft, it uploads automatically.
3.   Agents can request a read receipt when sending mail.
4.   Agents can send mail with large attachments — tens of megabytes work fine.
5.   Agents can manage mail templates — weekly reports and notifications don't need rewriting every time.
6.   Agents can share mail to group chats or DMs — sync important mail to a chat with one tap.
7.   Agents can attach a calendar invite to outgoing mail — send the invite and the email together.
8.   Agents can schedule mail for later — draft it, set a time, send automatically.
9.   Agents can recall sent mail within 24 hours — there's still time to fix a mistake.
10.   Agents can use personalized mail signatures — different signatures for different contexts.
11.   Agents can send from an alias — works for shared mailboxes or mailing groups.
12.   Agents can manage incoming mail rules — set them up once, mail sorts itself.
13.   Agents can listen for new mail in real time — get notified the moment something important arrives.
14.   Agents can read full mail threads — see the entire back-and-forth in one shot.

**Approvals**

1.   **Agents handle approvals six ways in one command** — query, approve, reject, transfer, withdraw, and cc.
2.   Agents can nudge pending approvals and review your own request history.

**Tasks**

1.   **Tasks can be assigned to AI agents** — your agent isn't just a tool, it's a teammate.
2.   Agents can pick up action items from Minutes or chat and create tasks — things mentioned in conversation don't get lost.
3.   Agents can search tasks, set parent-child relationships, group them, and subscribe to updates — the full toolkit for task management.
4.   Agents can add tasks to specific sections in a list — kept organized.
5.   Agents can upload attachments to tasks — finished work goes straight on the task.

**OKR**

1.   Agents can read and manage your OKRs — check goal progress without opening the OKR page.
2.   Agents can create, view, update, and delete OKR progress entries — weekly OKR updates without manual entry.

**Attendance**

1.   Agents can pull your lateness, missed clock-ins, and overtime records in one command — end-of-month review without opening the app.

### VI. Agents help you find things

Cross-domain search — docs, mail, and people, all easy to locate.

1.   Agents can search chat history — filter by keyword, sender, time range, or who was @-mentioned, even in long chats.
2.   Agents can find docs by what **you created, edited, viewed, or commented on** — no need to remember when or what kind.
3.   Agents can search Cloud Docs by keyword, edit time, or creator — fast even with thousands of docs.
4.   Agents support advanced search filters for docs — pinpoint what you need without sifting through results.
5.   Agents can search meeting recordings by keyword, attendees, or time — easy to find the right one even with hundreds.
6.   Agents can page through summary lists of mail — scan hundreds quickly.
7.   Agents can look up coworkers by name or email — multiple lookups in parallel.
8.   Agents can filter Contacts with criteria like "people you've chatted with" or "external contacts" — find the right person quickly.

### VII. Bring an agent to Lark — one command, 15 business domains ready

One command installs the CLI, creates the app, and handles authorization. Your agent gets every capability above right away — no doc reading, no webhook setup, no permission requests.

1.   MIT open source. Install with `npx @larksuite/cli@latest install` — **no build step, no environment setup.**
2.   **One-step install across platforms** — macOS, Linux, and Windows supported.
3.   **`lark-cli update`** updates the CLI in one command, with verification and rollback — revert in seconds if something breaks.
4.   The CLI checks for new versions on startup and tells you — no quietly running an outdated build.
5.   Add `--jq` to any command to extract just the fields you want — saves tokens, gives LLMs clean structured data.
6.   Real-time event subscriptions over WebSocket — respond to messages, calendar changes, and doc updates as they happen.
7.   **`lark-cli schema`** shows the parameters and permissions for any Lark API — no doc-switching during development.
8.   Companion **AI Agent Skills** library (`npx skills add larksuite/cli`) — wrap common operations as Skills, callable directly from AI coding tools like Claude Code.

### VIII. Agents in Lark run with safety built in

**Available through the open-source CLI**

1.   **Strict mode**: agent bot / user identity can be configured at a fine grain — no overreach, each identity does only what it's meant to.
2.   Files created by your agent are automatically permissioned for you — no more "I asked the agent to create it but I can't access it."

**Platform-level security and governance**

These apply when agents run within Lark's hosted pipeline. Some capabilities are available in commercial editions or specific deployment setups.

1.   Agent content goes through baseline safety checks — prompt injection protection and sensitive data redaction help keep AI-generated content in bounds.
2.   Agent behavior is monitored for risk — the platform watches key calls for risk and anomalies, helping prevent abuse.
3.   Agents access your resources within your own permissions — they can't see anything you can't see.
4.   Key agent actions are logged for audit — searchable trails make after-the-fact review possible.
5.   OAuth Device Flow + keyless app creation — secure onboarding without managing secrets.
